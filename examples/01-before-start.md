# Example — Appraise, before starting

Usage: *check whether something is worth starting.*

## Input

> "Thinking about spending tomorrow writing a script that auto-formats my
> messy meeting notes into clean structured summaries. Worth it?"
> Status: **possible**

## Read

**RAISE · Substance 5/6 · Signal 1/4 · sweet-spot: yes**

- **Substance (latent external value):** pain_density 2 (messy notes → clean
  summaries is a common, named pain, not just yours), frequency 2 (a pervasive
  workflow — almost everyone takes meeting notes), leverage 1 (broadly useful in
  principle, but *this* implementation is personal until shown to travel; scored
  down per the gate-flip rule). Sum 5 — clears the gate. Work like this is
  genuinely wanted in the world.
- **Signal (legibility):** only `self_starting` is surfaced. The build *would*
  involve a tradeoff, but as described it isn't articulated — that's latent, so it
  doesn't count. No frontier-tool angle shown, no future-state. As scoped it's
  invisible.
- **Sweet spot:** yes — "clean form to unstructured data" is your high-leverage
  lane. **But note:** the tip only fires at Borderline+. With Signal 1/4 this is a
  Raise, and the sweet-spot does **not** rescue it. Invisible stays invisible.

**Binding constraint:** Substance is strong; legibility is the problem. The
fastest lift isn't building *more* — it's building it *visibly*: surface the
tradeoff you'll hit (e.g. "chose recall over precision so I never lose an action
item, and accept some noise — here's the measured rate"), and show the AI
extraction step.

**Move — Raise:** build it, and scope in the one Signal dimension that's nearly
free given what you're already doing — the named tradeoff. That single addition
carries this from a private script toward a postable "here's a pattern I found
working with AI." Start it.
